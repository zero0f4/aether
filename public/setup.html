AETHER configuration

Welcome · first-run setup

AETHER is een passief reconnaissance- en observability-platform voor je UniFi-netwerk. Configureer hieronder de UniFi-controller en optionele intel-bronnen. Alle gegevens worden lokaal opgeslagen in .env (chmod 600); niets wordt naar derden verzonden behalve via expliciete intel-koppelingen die jij activeert.

UniFi controller vereist

▾
Maak in je UniFi-controller een local read-only admin: Settings → Admins & Users → Create New Admin, kies Restrict to local access only + rol View Only. Met read-only kan AETHER alleen lezen — geen wijzigingen mogelijk.

Server & data-retentie optioneel

▾
Tweede graads: poll-interval en hoelang historische data wordt bewaard in de lokale SQLite-database. Standaardwaarden zijn voor de meeste setups prima.

WiGLE.net wardrive lookup optioneel

▾
Met WiGLE-credentials kan AETHER elke neighbor-BSSID opzoeken in de wereldwijde wardrive-database — locatie, eerste/laatste-gezien, SSID-historie. Maak een gratis account aan op wigle.net/account → Show My Token → de "Encoded for use" base64-string decoderen → de userId:apiToken hieronder invullen.

Zigbee · Home Assistant koppeling optioneel

▾
Activeert de ZIGBEE-tab met live mesh-graph (coordinator, routers, end-devices, LQI per link), end-devices die clusteren rond hun parent-router, en een Zigbee-band-overlay op de SPECTRUM 2.4 GHz view zodat je in één oogopslag ziet of WiFi en Zigbee elkaar bijten.

Vereisten:
  • Home Assistant met de ZHA-integratie actief (Zigbee2MQTT wordt nog niet ondersteund)
  • HA bereikbaar vanaf deze server op http(s)://host:port
  • Een long-lived access token uit HA → klik op je profiel (linksonder) → tab Security → onderaan "Long-Lived Access Tokens" → Create token
AETHER vraagt alleen-lezen ZHA-data op (devices, neighbors, LQI). Geen schrijf-acties op je HA.

Toegangscontrole optioneel

▾
AETHER draait standaard open op localhost. Wil je het buiten je Mac toegankelijk maken (reverse-proxy / tailscale / LAN), zet dan tokens. Lege waarde = open. Tokens stuur je mee als ?token=… of Authorization: Bearer ….
